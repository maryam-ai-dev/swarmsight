How it works

Check before go-live. Control during work. Prove afterwards.

01

Test the agent

Run it against realistic cases in an arena. It earns a certificate that says exactly what it may do.

02

Certify the sources

Check documents are current and permissioned. Unsafe sources are blocked before an agent can cite them.

03

Control the action

Every action returns one plain verdict: allowed, masked, held for an officer, or blocked.

04

Prove the decision

Source, policy version, the draft, the human edit, and the approver are sealed in a verifiable trail.

What the agent is allowed to see

Sensitive fields are masked at the boundary, before the record reaches the agent. The agent cannot leak what it never received.

Masked means the agent knows a field exists but not its value. No access means the field is removed entirely, the agent never knows it was there.
How it is decided: a field reaches the agent only if the source's own permissions allow it and the department's sensitivity policy permits it at the agent's clearance. Both must agree. Every mask is recorded, so an auditor can prove the data was never exposed.
